Using $GOPATH effectively.

Using $GOPATH effectively can be confusing at first. If you follow guides on golang.org you’ll end up with way too many packages and dependency confusion in $HOME/go/src. This is the exact same problem new Python developers have before someone tells them about virtualenv. You end up making a mess out of the system Python and use sudo easy_install way too much.

However, we can ignore some of the fancy features that virtualenv provides. For instance, it doesn’t usually make sense to have different versions for Go unless you’re working on the language itself, in which case you probably know what you’re doing. All we really care about is having go install put packages in the right place and not clobber or clutter anything.

Since all we are doing is modifying the environment to get a new Go workspace, I wrote a simple bash script that does this for you. It creates the necessary directories for a Go workspace (pkg, src, bin), modifies the $GOPATH and $PATH variables and that’s it.

There are two versions, one spawns a subshell with these updated variables, found on the branch inve based on a gist about how virtualenv is doing it wrong and one that I find a bit more pleasant to use that you have to source and modify the current shell environment.
